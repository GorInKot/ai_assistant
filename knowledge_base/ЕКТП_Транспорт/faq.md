# FAQ: ЕКТП и транспорт

## Для чего используется ЕКТП?
ЕКТП используется для оформления заявок на служебный транспорт и отслеживания статуса согласования.

## Какие статусы у заявки на ТС?
Типовая цепочка статусов:
- `Черновик` -> `На обработке` -> `Принята` -> `Поручена` -> `ТС назначены` -> `В работе` -> `Успешное завершение`.

## Кто участники заявочного процесса?
Участники процесса:
- Участник 1: ЦДС (доступ и координация),
- Участник 2: специалист по транспорту филиала/ГО (справочники, договорно-тарифная база),
- Участник 3: ответственный сотрудник ПУ за подачу и заполнение заявок,
- Участник 4: подрядная организация по транспорту (назначает ТС/водителя, подтверждает/отклоняет факт),
- Участник 5: специалист по транспорту филиала/ГО (финальный акцепт, совпадает с Участником 2).

## Типовая ошибка при создании заявки
Если не указан корректный маршрут "откуда/куда", заявка не проходит первичную проверку.

## Что делать, если пробег по ГЛОНАСС не совпадает?
Проверить ПУ, часовой пояс и фактическое время, затем выполнить повторную ретрансляцию данных по инструкции. Если расхождение сохраняется, эскалировать вопрос в ЦДС.

## Ограничение
Срочные заявки могут требовать дополнительного согласования руководителя.
