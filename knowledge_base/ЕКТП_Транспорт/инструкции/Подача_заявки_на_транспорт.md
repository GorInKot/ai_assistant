# ЕКТП: подача заявки на транспорт (рабочая инструкция MVP)

## Назначение процесса
Процесс используется для заказа транспорта на выезды специалистов на объекты строительства, фиксации фактической работы и подготовки данных к акцепту/оплате услуг перевозчика.

## Участники процесса
- Участник 1: ЦДС (доступ и координация по ЕКТП).
- Участник 2: специалист по транспорту филиала/ГО (справочники, договорно-тарифная база, акцепт).
- Участник 3: ответственный сотрудник ПУ за подачу и заполнение заявок.
- Участник 4: подрядная организация по транспорту (назначение ТС/водителя, подтверждение/отклонение факта).
- Участник 5: специалист по транспорту филиала/ГО (совпадает с Участником 2, роль акцепта).

## 1) Получение доступа к ЕКТП
1. Сотрудник оформляет заявку на доступ по установленной форме.
2. Заявка направляется в службу, отвечающую за доступы.
3. После предоставления доступа сотрудник входит в систему и проверяет:
   - корректное ПУ;
   - корректный часовой пояс.

Связанный документ: `Шаблон заявки на ИС ЕКТП.docx`.

## 2) Подготовка к созданию заявки
Перед созданием заявок должны быть заполнены справочники:
- сотрудники;
- контрагенты;
- договоры и спецификации;
- производственные программы;
- иные общие справочники.

Если справочники не заполнены, часть полей заявки не будет корректно подбираться автоматически.

## 3) Создание заявки в ЕКТП
1. Открыть `Парк ТС` -> `Заявки на ТС`.
2. Нажать `+Новая` (новая заявка создается в статусе `Черновик`).
3. В поле ФИО выбрать контактное лицо.
   - Телефон и email подтягиваются из справочника сотрудников.
4. Поля `Внутренний заказчик` и `Номер заявки` заполняются автоматически.
5. Нажать `Добавить задание`.
   - В одной заявке может быть от 1 до 30 заданий.
   - 1 задание = 1 машина = 1 выезд.
6. Заполнить параметры задания:
   - тип заявки: обычно `Пассажиры`;
   - маршрут: точки на карте или шаблон из справочника маршрутов;
   - плановые дата/время начала и окончания;
   - тарифный блок: подрядчик, договор, спецификация, тариф, производственная программа;
   - количество ТС, количество пассажиров, тип ТС, комментарий.
7. Нажать `Сохранить` (можно на промежуточных этапах, статус останется `Черновик`).
8. Нажать `Разместить`, чтобы передать заявку на обработку.

## 4) Статусы заявки и переходы
- `+Новая` -> `Черновик`.
- `Разместить` -> `На обработке`.
- `Отклонить` (на этапе `На обработке`) -> `Черновик` для корректировки.
- `Принять` -> `Принята`.
- `Назначить` ответственного -> `Поручена`.
- `ТС назначены` -> `ТС назначены`.
- `В работу` -> `В работе`.
- `Завершить` -> `Успешное завершение`.

Примечание: легенда статусов и действий по заявке отображается в карточке заявки.

## 5) Взаимодействие с контрагентом
1. После назначения ответственного заявка уходит контрагенту.
2. Контрагент по каждому заданию:
   - либо назначает ТС и водителя;
   - либо отклоняет задание с указанием причины.
3. Если в срок по договору ТС не назначено:
   - связаться с контрагентом для ускорения;
   - при невозможности назначения использовать действие `Не назначено ТС`.
4. После назначения ТС поступает уведомление заказчику.
5. Если все корректно, нажать `ТС назначены`, затем `В работу`.

## 6) Ввод фактической работы по заданию
1. По завершении выезда зафиксировать в путевом листе:
   - время прибытия;
   - время окончания;
   - показания датчика/пробега.
2. В ЕКТП открыть блок `Фактическая работа`.
3. Выбрать режим ввода (рекомендуется `Указанные`).
4. Заполнить:
   - номер путевого листа;
   - фактические дата/время прибытия и убытия;
   - пробег и/или машино-часы (в зависимости от тарифа договора).
5. Проверить сравнение фактического пробега с пробегом по ГЛОНАСС:
   - целевое отклонение не более 5%.
6. Нажать `Завершить задание`.
7. После завершения всех заданий заявка считается выполненной.

## 7) Передача факта контрагенту и акцепт
1. После ввода фактических данных нажать `Передать` (по заданию).
2. Контрагент подтверждает или отклоняет переданные данные.
3. Пока данные не отклонены контрагентом, редактирование может быть ограничено.
4. После подтверждения и проверки выполняется акцепт заявки ответственным участником.
5. После акцепта работы могут быть переданы в оплату.

## 8) Работа с отклонениями и ретрансляцией
Если ГЛОНАСС-пробег равен 0 или отклонение более 5%:
1. Запросить повторную ретрансляцию данных по инструкции.
2. Обновить данные в задании.
3. Проверить корректность ПУ, часового пояса и фактического времени.
4. Если расхождение сохраняется, эскалировать вопрос в ЦДС.

Связанный документ: `1. Инструкция правильного алгоритма повторной отправки телематических данных...pdf`.

## 9) Заполнение справочников
Справочники находятся в разделе `Настройки`.

Базовый перечень:
- `Подразделения`;
- `Сотрудники`;
- `Контрагенты`;
- `Договоры`;
- `Общие справочники`.

## 10) Заполнение производственной программы (ПП)
1. Открыть `Парк ТС` -> `Производственная программа`.
2. Нажать `Добавить`.
3. Указать заказчика, подразделение и базовые параметры ПП.
4. Во вкладке `Договор и тарифы` выбрать договор и исполнителя, сохранить.
5. Через меню `...` добавить спецификации/тарифы по договору.
6. Во вкладке `Режим работы` добавить режим (по шаблону или новый).
7. Во вкладке `Расчет` выполнить расчет, сохранить изменения.

Важно: ПП создается после внесения контрагента и договора в справочники.

## 11) Что нужно доработать в следующей версии
- альтернативные сценарии и обработка ошибок (расширенный каталог);
- SLA по статусам и контроль сроков;
- расширенная матрица ответственности по филиалам/ПУ (в пределах уже заданных ролей участников).
