# ЦУС: проведение строительного контроля (актуальная инструкция MVP)

## Назначение
Процесс предназначен для ведения строительного контроля в ЦУС: от настройки объекта и проверок в модуле `Стройконтроль` до ведения журналов, формирования ИД, согласования и подписания документов.

## Основные процессы по ЦУС (строительный контур)
1. Получение доступа к ЦУС.
2. Подготовка карточки объекта и участников.
3. Проведение проверок в модуле `Стройконтроль`.
4. Ведение журналов в модуле `Журналы`.
5. Формирование исполнительной документации (ИД).
6. Согласование и подписание через `СЭД`.
7. Формирование отчетов/аналитики (при необходимости).

## Участники (рабочая модель MVP)
- Администратор/координатор доступа к ЦУС.
- Координатор объекта (заказчик/техзаказчик/ПТО).
- Инженер строительного контроля.
- Представитель подрядной организации.
- Согласующее/подписывающее лицо (ЭП).

## 1) Получение доступа к ЦУС
1. Оформить заявку на доступ по форме.
2. Передать заявку ответственному за доступ.
3. После выдачи доступа проверить:
   - доступ к объекту;
   - доступ к модулям `Информация`, `Стройконтроль`, `Журналы`, `ИД`, `СЭД`;
   - получение уведомлений.

Связанный документ: `Шаблон заявки на доступ в ЦУС.docx`.

## 2) Подготовка объекта и участников (модуль `Информация`)
1. Открыть карточку объекта.
2. На вкладке `Участники` добавить юридических и физических лиц.
3. Назначить роли участникам (влияет на автозаполнение журналов и актов).
4. Для физических лиц добавить документы о назначении (приказ/доверенность), если требуется.
5. На вкладке `Файловое хранилище` загрузить исходные файлы.

Важно: без корректных ролей участников автозаполнение реквизитов журналов и актов работает неполно.

## 3) Проведение проверок (модуль `Стройконтроль`)
### 3.1 Создание проверки
1. Открыть модуль `Стройконтроль`, вкладка `Проверки`.
2. Создать проверку и заполнить вкладку `Информация`.
3. Указать `Вид контроля качества` (например: входной, операционный, приемочный, инспекционный и др.).

### 3.2 Работа с чек-листами проверки
1. В карточке проверки перейти на вкладку `Чек-листы`.
2. Создать чек-лист вручную или на основе шаблона.
3. Зафиксировать результаты по пунктам чек-листа.

### 3.3 Работа с результатами проверки
1. По результатам проверки оформить сущности модуля:
   - акт проверки;
   - предписание;
   - недостаток;
   - акт устранения недостатков (после выполнения корректирующих действий).
2. При необходимости использовать вкладку `Вызов инспектора`.

## 4) Ведение журналов (модуль `Журналы`)
### 4.1 Создание журнала
1. Открыть модуль `Журналы` в карточке объекта.
2. Выбрать тип из шаблонного перечня и нажать `Добавить журнал`.
3. Заполнить поля (тип, номер, примечание) и нажать `Создать`.

Примечание: произвольные типы журналов не создаются, используются предустановленные карточки.

### 4.2 Заполнение журнала
1. В карточке журнала открыть вкладку `Реквизиты журнала`.
2. Нажать `Проставить реквизиты по умолчанию` (автозаполнение из модуля `Информация`).
3. При необходимости скорректировать реквизиты вручную.
4. На вкладке `Разделы`:
   - `Заполнить` для автоматического заполнения;
   - `Добавить` для ручного ввода.
5. Прикрепить файлы (ПК, файловое хранилище, буфер обмена).

### 4.3 Работа с замечаниями по журналу
1. Добавить замечание к журналу целиком или к отдельной записи.
2. Указать текст, ответственного, срок устранения.
3. Ответственный направляет ответ.
4. Проверяющий принимает ответ (`Принять`), статус замечания -> `Закрыто`.

### 4.4 Согласование и подписание журнала
1. Запустить ДО: `Типы ДО` -> `Согласование`.
2. После согласования статус ДО -> `Утверждено`.
3. Запустить `Подписание ЭЦП`.

## 5) Формирование исполнительной документации (ИД)
Исполнительные документы создаются:
- в модуле `Исполнительная документация`;
- из модуля `ГПР`;
- из модуля `Журналы` (через связку с записью журнала).

Базовый порядок:
1. Создать документ нужного типа (например, АОСР).
2. Проверить автоподстановку участников из модуля `Информация`.
3. Заполнить обязательные поля.
4. Указать/связать позиции ГПР (если требуется).
5. Прикрепить подтверждающие файлы.
6. Нажать `Сохранить` (черновик) или `Провести` (готов к согласованию).

Важно: для части актов добавлен контроль обязательных полей при проведении.

## 6) Согласование и подписание (ИД + СЭД)
1. Открыть проведенный документ.
2. На вкладке `Согласование` отправить документ на согласование (по шаблону/маршруту).
3. При замечаниях:
   - создать новую версию документа;
   - устранить замечания;
   - повторно направить на согласование.
4. На вкладке `Подписание` отправить документ на ЭП.
5. Контролировать итоговые статусы в карточке.

## 7) Использование модуля `СЭД`
Модуль применяется для движения документов:
- создание карточки документа;
- запуск документооборота по типу/регламенту/параметрам;
- контроль статусов (`Требует действия`, `Активные`, `Отправлены мной`);
- выгрузка листов согласования.

## 8) Ключевые статусы и переходы
- Замечание: `Активно` -> `Закрыто`.
- Журнал в ДО: после согласования -> `Утверждено`.
- Документ ИД: `Черновик` -> `Проведен` -> `Согласовано`/`Отклонено` -> `Подписан`.
- При возврате: `Вернуть на доработку` -> исправление -> повторное согласование.

## 9) Типовые ошибки и контрольные точки
- Не назначены роли участникам в модуле `Информация` -> некорректное автозаполнение.
- Пропущены обязательные поля акта/журнала -> проведение/маршрут блокируется.
- Замечания не закрыты -> процесс согласования может быть приостановлен.
- Для связей с ГПР фактический объем превышает допустимый -> документ не проводится.

## 10) Актуальные изменения версии
Ключевые обновления по новой версии ЦУС (в части стройконтура):
- в `Стройконтроль` добавлен `Вид контроля качества`;
- в карточке проверки добавлена вкладка `Чек-листы`;
- в `Журналы` добавлен импорт/экспорт через Excel-шаблон;
- в ИД расширены функции выбора значений из ранее введенных данных;
- усилен контроль заполнения обязательных полей для ряда актов при проведении;
- доработана логика последовательных уведомлений при согласовании.
